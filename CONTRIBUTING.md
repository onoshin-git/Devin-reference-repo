# 貢献ガイド

このリポジトリは **実装の正本ではなく、参考用のテンプレート集** です。  
Issue や PR を出すときは、まず「正本にしたいのか」「参考情報として残したいのか」を明確にしてください。

## 受け入れたい変更
- リファレンスの読みやすさを上げる改善
- 参考用 stub やテンプレートの補強
- Mermaid 図や契約書の分かりやすさ向上
- 既知の認識ずれを防ぐための説明追加
- 新人向けの手引きやレビュー観点の追加

## 受け入れない変更
- `references/` や `old/` の内容を正本扱いする変更
- この repo 単体で本番 deploy できると誤認させる記述
- 機密情報、個人情報、実環境の秘密値の追加
- 実際の業務リポジトリ固有コードの丸ごと転載

## 変更時のルール
1. `README.md` と `AGENTS.md` の「参考用である」前提を崩さない
2. `authoritative` と `reference` を混同しない
3. 追加ファイルには用途を明記する
4. Mermaid 図を追加する場合は対応する Markdown も更新する
5. JSON stub は構文が壊れていない状態で追加する

## PR に含めてほしい内容
- 変更の目的
- どの利用シーンで役立つか
- 参考情報か、運用ルールか、テンプレートか
- 既存ファイルへの影響
- 必要なら before / after

## レビュー観点
- 初見の人が誤解しないか
- Devin や Codex に読ませたときに reference を正本と誤認しないか
- 画面、API、データ、deploy のどの観点を補強したのかが明確か
- 将来の handoff や RAG に転用しやすいか
